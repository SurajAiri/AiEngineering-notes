## Note Format

Each topic follows a consistent structure:

- **Concepts** — What it is and why it matters
- **Diagrams** — ASCII diagrams to visualize the concept
- **Simple Code** — Minimal code to understand the idea without boilerplate
- **Production Code** — Complete, runnable implementations
- **Pitfalls & Common Mistakes** — What to avoid
- **Trade-offs** — Decision framework for real-world choices
- **Key Takeaways** — Summary of the most important points
